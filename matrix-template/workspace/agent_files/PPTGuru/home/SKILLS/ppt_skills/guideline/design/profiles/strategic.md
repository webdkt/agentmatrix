# Strategic

> Applicable scenarios: Corporate strategic planning, board presentations, fundraising roadshows, investor pitches, M&A presentations, ESG reports, long-term vision planning, etc.
> Style anchor: McKinsey/BCG strategy decks + Goldman Sachs investor presentations + Fortune 500 board materials

## Design Philosophy
- **Executive gravitas**: Design must convey authority, maturity, and strategic depth; avoid anything that appears superficial or trendy
- **Data-driven narrative**: Every strategic claim must be backed by data, market analysis, or logical framework; visuals serve evidence
- **Long-term perspective**: Layouts and visuals should feel timeless, not fashionable; classic, enduring design choices
- **Clarity under complexity**: Strategic topics are inherently complex; the design's job is to simplify and structure, never to add noise
- **Information density: High (85-95%)**: Board and investor audiences expect comprehensive, data-rich materials; minimalism is a liability

## Chapter Page Style
- **Preferred**: Dark background chapter pages (`$primary` or `$darkBg` fill) with accent strip, "CHAPTER" label, and semi-transparent white watermark number
- **Rationale**: Dark chapter pages signal gravitas and create visual rest points in dense strategic decks. The contrast between dark chapter pages and light content pages improves navigation rhythm.
- **Alternative**: If the user explicitly requests a lighter tone, light chapter pages are acceptable.

## Text-to-Visual Ratio: Balanced, Data-Anchored
- Text carries the strategic argument and narrative; visuals (charts, matrices, frameworks) provide the analytical scaffolding
- Recommended ratio: approximately 55% text + 35% analytical visuals (charts, matrices, diagrams, tables) + 10% structural whitespace
- Frameworks and models (e.g., SWOT, BCG Matrix, Porter's Five Forces) should be built with shapes and text, not imported as static images
- Financial and market data must be presented with precision and clear sourcing

## Color Guidance
- **Conservative, premium palette**: Strategic presentations demand colors that signal stability, trust, and sophistication
- **Single primary color with depth**: Choose one strong primary color (deep navy, forest green, burgundy, charcoal) and use it consistently; variations in shade provide hierarchy
  - Navy/Charcoal: Authority, stability, consulting tradition (McKinsey/BCG anchor)
  - Forest Green: Growth, sustainability, ESG focus
  - Burgundy/Maroon: Heritage, premium, private equity gravitas
  - Deep Teal: Modern strategy, tech-forward thinking
- **Neutral dominance**: White or very light gray backgrounds; near-black or charcoal text; gray hierarchy for secondary information
- **Restrained accent**: One accent color (copper, muted gold, subtle orange) used sparingly for key data highlights, critical callouts, or strategic imperatives
- **Strictly prohibit**: Bright, playful colors; rainbow palettes; gradients on backgrounds (subtle gradients on chart fills may be acceptable if extremely restrained); neon or high-saturation tones
- **Chart and data colors**: Use primary color family + grays; one accent color for the "key metric" or "target" series; avoid multi-color chaos

## Font Guidance
- **Titles and headings**: Strong serif (e.g., Times New Roman, Noto Serif SC) for tradition and authority, or bold geometric sans-serif (e.g., Montserrat Bold, Arial Black) for modern strategy firms
- **Body text and data**: Clean, highly legible sans-serif (e.g., Arial, Calibri, MiSans); numbers and financial data should use tabular figures if available
- **Chinese**: Microsoft YaHei or MiSans for body; Songti or Noto Serif SC for formal headings
- Font size hierarchy should project confidence and readability:
  - Cover title: 44-52px
  - Executive summary / vision statement: 32-40px
  - Page titles (Action Titles): 26-32px
  - Body text: 18-22px (use 22px for lighter pages, 18px for dense analytical pages; must not go below 18px)
  - Data labels, chart annotations, footnotes: 12-14px
  - Minimum font size: 11px (only for dense table footnotes or source annotations)

## Content Page Structure
- **Strategic page structures**:
  - Action Title (conclusion headline — mandatory for strategy decks)
  - Supporting framework / model / matrix
  - Evidence layer (data, charts, case evidence)
  - Source and footnote area
  - Page numbers
- **Common layouts**:
  - Executive summary: Key conclusions + 3-5 supporting bullets
  - Market analysis: Market sizing chart + competitive landscape text
  - Strategic framework: Custom-built matrix or model using shapes + text
  - Financial projection: Data table + key assumption callouts
  - Implementation roadmap: Timeline or Gantt-style chart + milestone descriptions
  - Risk matrix: Probability-Impact matrix with categorized risks
- **Framework emphasis**: Strategy decks rely heavily on consulting-style frameworks; build these with geometric shapes, clean lines, and precise typography

## Narrative Style: Pyramid Principle / Logic-Driven
- **Action Titles are mandatory**: Every content page title must be a complete conclusion sentence (e.g., "Market consolidation creates 3x growth opportunity" not "Market Analysis")
- **SCQA structure** (Situation-Complication-Question-Answer) for problem-solving sections
- **MECE principle**: Mutually Exclusive, Collectively Exhaustive — structure pages and sections to show comprehensive, non-overlapping analysis
- **So what? test**: Every data point and chart must connect to a strategic implication; raw data without interpretation is forbidden
- **Story arc**: Current State -> Strategic Challenge -> Analysis -> Strategic Options -> Recommended Strategy -> Implementation -> Financial Impact
- **Strategic planning narrative framework**: Background & Trends → Problems & Opportunities → Strategic Objectives → Implementation Path → Resource Requirements → Expected Returns & ROI → Risks & Safeguards → Summary & Call to Action
- **Fundraising narrative framework**: Problem → Solution → Product → Market Size → Business Model → Competitive Advantage → Traction → Team → Financial Projections → Funding Requirements
- The two frameworks can be flexibly selected or blended based on the scenario
- **Expected returns must be explicitly stated**: Every strategic initiative must be clearly linked to quantified returns, enabling decision-makers to see the input-output ratio
- Language is confident, powerful, and restrained — not exaggerated but willing to take a stance; data has sources
- **Data source annotations**: When citing market data or industry reports, source notes should be appended (e.g., `Source: Institution Name, Year`), using `<a href="url">` hyperlinks to point to the original report or data page for enhanced verifiability

## Content Expression Techniques
- **Consulting-style frameworks**: Build matrices, 2x2 grids, Porter's frameworks, flywheel models using shapes and text boxes (never import as images)
- **Big number callouts**: Key financial metrics (TAM, revenue, EBITDA, growth rate) displayed as large numbers + brief context
- **Waterfall and cascade charts**: Perfect for showing financial build-ups, cost breakdowns, or strategic option comparisons
- **Scenario analysis**: Side-by-side scenario tables (Base/Upside/Downside) with key drivers
- **Roadmaps and timelines**: Linear or phased timelines showing strategic milestones and interdependencies
- **Heat maps and matrices**: Color-coded tables or grids for risk, priority, or capability assessments
- **Source annotations**: Every data point and external claim must cite the source; format: `Source: [Institution, Year]` with `<a href="url">` hyperlinks where possible
- **Footnotes**: Use superscript numbering for assertions that require detailed sourcing; collect footnotes at the bottom of each page
- **TAM/SAM/SOM**: Market size visualized with nested circles (shape: ellipse) or 3 big numbers with definitions
- **Growth curves**: Traction data shown with line charts displaying growth trends, annotated with key milestones
- **Unit economics model**: Big numbers + formula decomposition (CAC, LTV, gross margin, etc.)
- **Competitive matrix**: Two-dimensional matrix chart positioning the competitive landscape, highlighting your advantageous position
- **Objective-Path-Returns chain**: Flowcharts or three-part structure clearly showing "What to do → How to do it → What you get"
- **ROI analysis**: Input-output ratio prominently displayed as big numbers, supplemented by cost breakdown and returns projection charts
- **Milestone roadmap**: Timeline showing key nodes, each annotated with phase objectives and expected output
- **Risk matrix**: Two-dimensional matrix (probability × impact) showing risk distribution, paired with mitigation strategies
- **Team showcase**: Photo + name + background (one line), highlighting the core team's industry experience and strategic execution capability
- **Funding plan**: Clear amount, allocation proportions (pie chart applicable), key milestone timeline
- **Client logo wall**: Logos of existing clients/partners displayed to enhance credibility

### Predefined Frameworks for Strategic Decks
When the content fits, use these proven consulting frameworks directly:

| Framework | When to Use | Visual Structure |
|-----------|-------------|-----------------|
| **TAM/SAM/SOM** | Market sizing slides | 3 nested circles (shape: ellipse) or 3 big numbers with definitions |
| **Unit Economics** | Business model / profitability | Big numbers row: CAC, LTV, LTV/CAC ratio, Payback period |
| **Competitive Matrix** | Positioning against competitors | 2×2 or N×M table; position "us" with accent color highlight |
| **Objective-Path-Returns** | Investment / project proposals | Top: objective; middle: 3-4 implementation paths; bottom: quantified returns |
| **Risk Matrix** | Risk assessment | 2×2 grid (Probability × Impact); color-code quadrants |
| **Milestone Roadmap** | Implementation planning | Horizontal timeline with chevron shapes; each phase has date + deliverable + owner |
| **SWOT / BCG Matrix** | Strategic analysis | 2×2 quadrants; use shape fills to differentiate categories |
| **Porter's Five Forces** | Industry analysis | Central circle (industry attractiveness) + 5 surrounding force boxes |
| **Growth Flywheel** | Self-reinforcing growth model | Circular loop of 4-6 interconnected drivers |
| **Waterfall Chart** | Financial bridge (e.g., revenue bridge) | Build-up bars from baseline to target using stacked rect shapes |

> **Rule**: Do not invent new frameworks when a standard consulting framework fits the content. Reusing recognized frameworks increases credibility with executive audiences.

## Image Usage Rules
**Allowed (analytical/strategic images)** — Images that carry strategic information:

| Image Type | Examples | Applicable Pages |
|---------|------|---------|
| Corporate/brand imagery | Company headquarters, flagship products, team leadership | Cover, company overview |
| Market/regional maps | Geographic market coverage, regional heat maps | Market analysis, expansion strategy |
| Product/service photos | Key offerings discussed in the strategy | Product strategy, portfolio analysis |
| Partnership/transaction logos | M&A targets, partner logos, JV entities | Deal structure, partnership pages |
| Timeline/event photos | Historical milestones, acquisition photos | Company history, transformation journey |

**Prohibited (decorative images)**:
- Stock photos of business people shaking hands, generic "success" imagery
- Abstract backgrounds, decorative textures, or atmospheric photography unrelated to strategy
- Unsourced data chart screenshots (build charts natively using chart elements)
- Decorative icons without analytical purpose

## Decoration Guidelines (Restrained)
| Element | Guideline |
|--------|---------|
| Geometric shapes | Allowed for frameworks, matrices, and structural layout; use primary color or gray fills |
| Thin lines and dividers | 1px solid lines in gray or primary color for separating sections and building frameworks |
| Subtle fills | Very light gray or primary-tint backgrounds for table rows or matrix cells |
| Minimal rounding | 0-4px corner radius at most; sharp corners convey precision |

## Decoration Prohibitions
| Prohibited | Alternative |
|--------|---------|
| Shadows | Flat design only |
| Gradient fills | Solid fills or very subtle tints |
| Decorative icons | Only use icons if they serve a structural or labeling purpose in frameworks |
| 3D effects, bevels, reflections | Strictly 2D, flat design |
| Rounded cards with heavy padding | Tight, precise layouts; use lines and whitespace for separation |
| Decorative page transitions or animations | Static, print-ready layouts |
| Unsourced data or assertions | Every claim must be traceable to a source |
| Long text paragraphs | Refine into bullet points and data |
| Unsupported claims without data | Attach specific numbers and sources |
| Overly flashy visual decoration | Clean, steady static display |
| Generic stock photos | Actual product screenshots or high-quality brand images |
| Information overload (>7 points per page) | Split pages; one focus per page |
| Vague competitive analysis | Specific metric comparison matrices or tables |
| Returns described only qualitatively | Quantified return figures (ROI, growth rate, cost savings) |
| Data piled without conclusions | Three-part structure: data + insight + action recommendation |
| Low-quality gradients and shadows | Solid colors or very subtle gradients to maintain premium feel |

## Additional Strategic-Specific Guidelines
- **Confidentiality markers**: If applicable, add "Confidential" or "Draft" watermarks in a subtle, professional manner
- **Appendix discipline**: Move detailed models, sensitivity analyses, and supporting data to appendix pages; keep main deck focused on conclusions
- **Financial precision**: All financial figures must include units (M, B, %); define acronyms on first use; ensure consistency in number formatting
- **Executive audience sensitivity**: Assume limited attention span; front-load conclusions; use appendices for detail; every page should pass the "elevator test" (comprehensible in 30 seconds)
- **Competitive discretion**: When discussing competitors, use factual, measured language; avoid hyperbole or unsupported claims
