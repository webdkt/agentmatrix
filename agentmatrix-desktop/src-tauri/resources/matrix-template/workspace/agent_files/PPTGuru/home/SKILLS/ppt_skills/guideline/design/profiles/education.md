# Education

## Design Philosophy
- **Pedagogical clarity first**: Every visual choice should enhance comprehension and retention; avoid cognitive overload
- **Structured progression**: Clear step-by-step logic that guides learners from known to unknown
- **Friendly and approachable**: Professional but not intimidating; warm and inviting without being childish
- **Visual scaffolding**: Use diagrams, icons, and color coding as cognitive aids to structure information
- **Moderate information density (60-70%)**: Balance content richness with breathing room; younger audiences need more whitespace

## Chapter Page Style
- **Preferred**: Light background chapter pages with colorful but soft watermark numbers
- **Rationale**: Educational content should feel accessible and friendly. Dark chapter pages can feel too formal or intimidating for learners.
- **Encouraged**: Use the chapter transition as a cognitive reset — include a simple icon or diagram previewing the upcoming section's topic.

## Text-to-Visual Ratio: Balanced, Visuals as Learning Aids
- Visual elements (diagrams, icons, illustrations, charts) serve as pedagogical tools, not mere decoration
- Recommended ratio: approximately 50% text + 40% visuals (diagrams, charts, icons, illustrations) + 10% interactive whitespace
- Key concepts should be reinforced with visual metaphors or diagrams
- Step-by-step processes must be visually segmented (numbered steps, flowcharts, timeline layouts)

## Color Guidance
- **Warm, inviting palette**: Choose 1-2 friendly primary colors (e.g., teal, orange, soft blue, sage green) that evoke curiosity and comfort
- **Color as cognitive coding**: Use consistent color coding for categories (e.g., all "Key Concepts" in one color, "Examples" in another, "Exercises" in a third)
- **High contrast for readability**: Ensure sufficient contrast between text and backgrounds for diverse classroom lighting conditions
- **Accent colors for engagement**: Use a brighter accent color sparingly for callouts, important notes, and interactive elements
- **Background strategy**: White or very light tinted backgrounds for content pages; richer colors reserved for chapter transitions and covers
- **Avoid**: Dark backgrounds for text-heavy pages (causes eye strain), neon or overly saturated colors, rainbow palettes that distract from content

## Font Guidance
- **Titles and headings**: Friendly sans-serif (e.g., Poppins, Montserrat, MiSans) with moderate weight — approachable authority
- **Body text**: Highly readable sans-serif (e.g., Open Sans, MiSans, Microsoft YaHei); if targeting children, consider rounded fonts
- **Chinese**: MiSans or Microsoft YaHei for modern education; rounded fonts for younger audiences
- Font size hierarchy should prioritize readability in classroom/lecture hall settings:
  - Cover title: 44-52px
  - Chapter/section titles: 32-40px
  - Page titles: 24-28px
  - Body text: 20-24px (must not go below 18px; for large lecture halls, use 24px+)
  - Captions, labels, notes: 16-18px
  - Minimum font size: 14px (only for non-essential annotations)

## Content Page Structure
- **Pedagogical page structures**:
  - Learning objective statement at the top of content pages
  - Core concept definition + visual explanation
  - Example / demonstration area
  - Practice / review / summary section
- **Common layouts**:
  - Left concept + right example/diagram
  - Top theory + bottom application
  - Step-by-step vertical progression (numbered, with icons)
  - Comparison/contrast side-by-side
  - Full-page diagram with callout annotations
- **Engagement elements**: "Think about it" boxes, "Key Takeaway" highlights, "Common Mistake" warnings — styled consistently with light backgrounds and left-border accents

## Narrative Style: Learning-Objective-Driven
- Begin with clear learning objectives ("By the end of this section, you will be able to...")
- Structure content as: What -> Why -> How -> Practice/Apply
- Use scaffolding: start simple, add complexity gradually
- Include regular checkpoints: summaries, review questions, or "pause and think" moments
- Connect abstract concepts to concrete examples and real-world applications
- Use inclusive, encouraging language; avoid overly technical jargon without explanation

## Content Expression Techniques
- **Concept maps and mind maps**: Visual hierarchies showing relationships between ideas
- **Step-by-step breakdowns**: Numbered sequences with icons; each step in a dedicated visual block
- **Annotated diagrams**: Use leader lines, callout boxes, and color coding to explain complex visuals
- **Before/After comparisons**: Side-by-side layouts for showing transformations or problem-solving
- **Interactive-style elements**: "Quiz" slides, "True/False" check slides, "Fill in the blank" styled text boxes
- **Highlighting and coding**: Use background color blocks, bold, and icon badges to mark: Definition, Theorem, Example, Important, Note, Exercise
- **Formulas**: Present with ample space; use color to highlight variables and operations
- **Tables**: Use for comparing methods, properties, or vocabulary; keep rows spaced generously

## Image Usage Rules
**Allowed (pedagogical images)** — Images that directly support learning:

| Image Type | Examples | Applicable Pages |
|---------|------|---------|
| Concept illustrations | Diagrams, infographics, process visualizations | Concept explanation pages |
| Real-world photos | Historical photos, nature/science photos, cultural images | Introduction, example pages |
| Student work examples | Correct/incorrect examples, student project screenshots | Evaluation, practice pages |
| Apparatus and tools | Scientific equipment, software interfaces, instrument close-ups | Lab, methodology pages |
| Character/avatar illustrations | For younger audiences: friendly mascots or guided characters | Elementary education, cover pages |

**Prohibited (decorative images)** — Images that do not support learning:
- Unrelated stock photos of people smiling or generic "education" imagery
- Abstract decorative backgrounds that reduce text readability
- Overly complex decorative illustrations that distract from content
- Unattributed images that violate copyright or academic integrity

## Decoration Guidelines (Allowed with Restraint)
| Element | Guideline |
|--------|---------|
| Icons | Allowed and encouraged for visual anchors; use consistent style (solid or outline), monochrome or single-color |
| Color blocks | Allowed for categorization and emphasis; use light tints (opacity 10-20%) |
| Rounded corners | Allowed (8-16px) for a friendly, modern feel; do not overuse |
| Light shadows | Minimal, soft shadows (2-4px blur, very light) permitted for cards/containers only |
| Decorative lines/dividers | Simple 1-2px lines in theme colors; use to separate sections |

## Decoration Prohibitions
| Prohibited | Alternative |
|--------|---------|
| Heavy drop shadows | Flat design or minimal shadows |
| Gradient backgrounds on content pages | Solid white or very light tinted backgrounds |
| Animated GIFs or motion effects | Static, clear visuals |
| Overly complex 3D diagrams | Clean 2D diagrams with clear labels |
| Decorative clip art unrelated to content | Purposeful icons or diagrams |
| Dense, wall-of-text layouts | Chunked content with visual breaks |

## Additional Education-Specific Guidelines
- **Audience age adaptation**: 
  - Elementary: Larger fonts (24px+ body text), more visuals, simpler layouts, friendly colors
  - Middle/High school: Balanced text and visuals, structured note-taking layouts
  - Adult/Professional: Higher density, data-driven, case-study focused, Action Titles encouraged
- **Accessibility**: Ensure color choices are colorblind-friendly; do not rely solely on color to convey meaning; add patterns or labels
- **Print-friendliness**: If students may print slides, test layouts on grayscale and avoid dark backgrounds
- **Pacing indicators**: Consider adding subtle progress indicators or section markers so learners know where they are in the lesson
