# Work Report

> Applicable scenarios: Annual work summaries, quarterly reports, OKR reviews, project retrospectives, team achievement showcases, performance reports, etc.
> Style anchor: Corporate internal reporting (Alibaba, Tencent internal templates) + modern SaaS dashboard aesthetics

## Design Philosophy
- **Results-oriented**: The design must foreground achievements, metrics, and outcomes; process is secondary to results
- **Clear hierarchy**: Readers (managers, executives) scan quickly; information must be scannable with clear visual priority
- **Professional positivity**: Celebrate achievements without appearing boastful; acknowledge challenges with constructive framing
- **Data celebration**: Numbers, charts, and progress indicators are the heroes of work reports
- **Information density: Moderate to High (70-85%)**: More dense than promotion, less dense than strategy; balance comprehensiveness with scannability

## Text-to-Visual Ratio: Data-forward, Text as Context
- Data visualizations (charts, progress bars, metric cards) carry the primary message; text provides context, interpretation, and narrative
- Recommended ratio: approximately 40% text + 45% data visuals (charts, tables, metric displays) + 15% whitespace/structure
- Use visual metaphors for progress (gauges, progress bars, trend arrows) to make achievement tangible
- Comparison is key: year-over-year, quarter-over-quarter, target vs. actual, team vs. team

## Color Guidance
- **Energetic yet professional**: Work reports can be more colorful than strategy decks but must remain corporate-appropriate
- **Primary color tied to company brand**: Use the company's official brand color as primary; if unavailable, choose a modern, confident color
  - Tech companies: Bright blue, teal, electric purple
  - Traditional industries: Navy, forest green, burgundy
  - Creative teams: Coral, mustard, vibrant gradients (used sparingly)
- **Success/positive indicators**: Use green or brand primary for positive metrics, achieved goals, and upward trends
- **Warning/challenge indicators**: Use amber or muted red for missed targets, risks, or areas needing attention (use sparingly and constructively)
- **Neutral backgrounds**: White or very light gray for content pages; brand color reserved for covers, chapter pages, and accent bars
- **Chart colors**: Primary brand color + 2-3 harmonious secondary colors + gray for neutral/comparison series

## Font Guidance
- **Titles and metrics**: Bold, modern sans-serif (e.g., Montserrat, Poppins Bold, MiSans Bold) for energy and clarity
- **Body text and descriptions**: Clean sans-serif (e.g., Open Sans, MiSans, Microsoft YaHei) for readability
- **Numbers and data**: Monospace or tabular sans-serif if available (e.g., Arial, Roboto) for alignment; ensure large metric numbers are prominent
- **Chinese**: MiSans or Microsoft YaHei for modern corporate feel; use bold weight for hierarchy
- Font size hierarchy should emphasize metrics and scannability:
  - Cover title: 44-52px
  - Big metric numbers (KPIs): 48-64px (oversized, as visual focal points)
  - Section/chapter titles: 32-40px
  - Page titles: 24-28px
  - Body text: 18-22px (must not go below 18px)
  - Chart labels, table text: 14-16px
  - Minimum font size: 12px (annotations and source notes)

## Content Page Structure
- **Work report page structures**:
  - Summary/KPI dashboard: 3-6 big metrics with trend indicators and brief context
  - Progress tracking: Goals/targets vs. actuals with progress bars or gauge charts
  - Achievement highlights: Key wins with supporting data and visuals
  - Project retrospective: Timeline + milestones + outcomes
  - Team/individual contributions: Attribution and recognition layouts
  - Challenges and next steps: Honest assessment + action plan
- **Common layouts**:
  - Top: Big metric row (3-4 numbers) | Bottom: Supporting charts or explanations
  - Left: Progress/chart | Right: Interpretation and action items
  - Full-page dashboard: Grid of metric cards (2x2 or 3x2)
  - Timeline: Horizontal or vertical project milestone timeline
  - Comparison table: Planned vs. Actual, Q1 vs. Q2, Team A vs. Team B

## Narrative Style: Achievement-Narrative + Forward-Looking
- Structure: What we set out to do -> What we achieved -> How we did it -> What we learned -> What's next
- Celebrate wins with data: "Achieved 127% of Q3 revenue target ($12.7M vs. $10M target)"
- Frame challenges constructively: "Delivery timeline extended by 2 weeks due to [reason]; mitigation: [action]"
- Include team attribution: "Led by [Team/Individual], contributing [X]% of total output"
- End with clear next-quarter or next-year objectives and resource needs
- Avoid defensive or apologetic language; be factual and forward-looking

## Content Expression Techniques
- **Big number / metric cards**: Oversized numbers (48-64px) + unit label + brief description + trend arrow/color indicator
- **Progress indicators**: Horizontal progress bars, circular gauges, or donut charts showing % completion toward goals
- **Trend sparklines**: Small inline line charts next to metrics showing recent trajectory
- **Comparison tables**: Clean tables with conditional formatting (green for ahead, red/amber for behind)
- **Bar and column charts**: Standard for comparing metrics across periods, teams, or categories
- **Timeline/roadmap views**: Show project phases, milestones, and delivery dates; mark completed vs. upcoming
- **Before/After comparisons**: Side-by-side metrics or visuals showing transformation or improvement
- **Milestone badges**: Visual badges or icons for achieved certifications, launches, or awards
- **Data tables with heat coding**: Use background cell colors (very light tints) to highlight top performers or areas of concern

## Image Usage Rules
**Allowed (work/evidence images)** — Images that document work and achievements:

| Image Type | Examples | Applicable Pages |
|---------|------|---------|
| Product/launch screenshots | New feature screenshots, app interfaces, website launches | Achievement, product pages |
| Team/event photos | Team offsite photos, launch event photos, award ceremonies | Team, culture, celebration pages |
| Process/operations photos | Warehouse improvements, office renovations, equipment upgrades | Operations, infrastructure pages |
| Data dashboard screenshots | Real analytics dashboards, monitoring tools (if not reproducible as charts) | Metrics, analytics pages |
| Certificate/award images | Partnership certificates, industry awards, patent documents | Recognition, credentials pages |

**Prohibited (decorative images)**:
- Generic stock photos unrelated to actual work performed
- Decorative backgrounds or textures
- Unrelated personal or lifestyle imagery
- Low-quality or unprofessional photos

## Decoration Guidelines (Allowed with Moderation)
| Element | Guideline |
|--------|---------|
| Metric cards / containers | Rounded rectangles (8-16px radius) with light fills or subtle borders to group metrics |
| Progress bars | Solid fills in brand/success colors; clean, rectangular or slightly rounded ends |
| Icons for categories | Solid, monochrome icons (16-24px) to label metric categories or sections |
| Color-coded badges | Small, solid-color pills or badges for status labels (e.g., "Completed", "On Track", "At Risk") |
| Subtle separators | 1px light gray lines or 8-16px whitespace gaps between sections |

## Decoration Prohibitions
| Prohibited | Alternative |
|--------|---------|
| Heavy shadows and 3D effects | Flat or minimal shadow (2px, very light) only on metric cards |
| Gradient backgrounds on content pages | Solid white or very light gray; gradients only on cover/chapter pages if brand-appropriate |
| Excessive decorative shapes | Clean, grid-aligned layouts; let data be the decoration |
| Animated or interactive elements | Static layouts optimized for PDF export and email distribution |
| Dense, unreadable tables | Chunk tables, use conditional formatting, and add white space between rows |
| Inconsistent color coding for status | Define a strict color system (e.g., Green=On Track, Amber=At Risk, Red=Behind) and apply consistently |

## Additional Work Report-Specific Guidelines
- **Quantify everything possible**: "Improved efficiency" is weak; "Reduced processing time by 34% (from 45min to 30min)" is strong
- **Template consistency**: Work reports are often recurring (weekly/monthly/quarterly); establish a repeatable template structure so readers know where to find information
- **Executive summary page**: Start with a 1-page summary containing the 3-5 most critical metrics and conclusions; many readers will only see this page
- **Comparison context**: Always provide benchmarks — targets, previous periods, industry averages, or competitor baselines
- **Attribution and recognition**: Include specific team or individual contributions; this builds morale and accountability
- **Forward-looking appendix**: End with a clear Next Steps / Q[X] Plan page with owners, deadlines, and success criteria
