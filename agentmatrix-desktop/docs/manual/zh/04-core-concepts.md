# 核心概念

理解以下概念将帮助您更有效地使用 AgentMatrix Desktop。

## Matrix World

**Matrix World** 是您的工作空间 —— 计算机上的一个目录，包含所有智能体配置、对话历史、生成的文件和系统设置。

当您设置 AgentMatrix（通过向导或手动方式）时，您选择一个目录作为 Matrix World。在其内部，一个隐藏的 `.matrix/` 目录存储所有系统数据：

```
MatrixWorld/
├── .matrix/
│   ├── configs/          # 所有配置文件
│   │   ├── agents/       # 智能体定义
│   │   ├── prompts/      # 系统提示词模板
│   │   ├── system_config.yml
│   │   ├── matrix_config.yml
│   │   ├── email_proxy_config.yml
│   │   └── llm_config.json
│   ├── sessions/         # 对话历史
│   ├── files/            # 智能体生成的文件
│   └── logs/             # 运行日志
└── (其他工作空间文件)
```

## 智能体 (Agent)

**智能体**是一个自主 AI 实体。每个智能体具有：

- **名称** —— 唯一标识符（如 "mark"、"SystemAdmin"）
- **人格 (Persona)** —— 定义智能体思考和响应方式的角色描述
- **技能 (Skills)** —— 智能体可以使用的能力（网页搜索、文件管理、邮件等）
- **类 (Class)** —— 决定行为模式的智能体类型
- **后端模型** —— 智能体使用的 LLM/SLM 配置

AgentMatrix 自带预配置的智能体：

| 智能体 | 角色 | 技能 |
|--------|------|------|
| **User** | 代表系统中的您 | base, email |
| **SystemAdmin** | 管理系统配置和智能体档案 | base, email, agent_admin, system_admin, file |
| **mark** | 网络研究和 OSINT 专家 | file, web_search |

您可以通过在 agents 目录中添加 YAML 配置文件来创建额外的智能体。

## 会话 (Session)

**会话**是一个对话线程 —— 您与一个或多个智能体之间持续的消息（邮件）交换。会话将您的工作组织成逻辑分组。

每个会话包含：

- 参与者列表（您和涉及的智能体）
- 邮件序列（对话历史）
- 未读标记，显示新消息

当您向智能体发送第一封邮件，或智能体主动发起联系时，会话会自动创建。

## 邮件 (Email)

**邮件**是会话中的一条消息。尽管名为邮件，AgentMatrix 中的邮件不会通过互联网发送（除非您配置了邮件代理）。它们是您和智能体之间的内部消息。

每封邮件包含：

- **发件人** —— 发送者（您或智能体）
- **收件人** —— 接收者
- **主题** —— 话题行
- **正文** —— 消息内容（支持 Markdown 格式）
- **附件** —— 可选的文件附件

邮件是您与智能体交互的主要方式 —— 您通过邮件发送任务，智能体处理后通过邮件回复结果。

## LLM 和 SLM

AgentMatrix 使用两种 AI 模型：

- **LLM（大语言模型）** —— "大脑"。用于复杂推理、规划和决策的强大模型。例如：GPT-4o, Claude Sonnet 4, Gemini 1.5 Pro。
- **SLM（小语言模型）** —— "小脑"。更快、更便宜的模型，用于邮件解析、分类和快速响应等轻量级任务。例如：GPT-4o Mini, Claude Haiku 4, Gemini 2.0 Flash。

这种分离确保复杂任务获得最佳 AI 质量，同时简单任务保持快速和经济。

## 技能 (Skill)

**技能**是智能体可以使用的能力。技能在智能体的配置中定义，决定了智能体可以访问哪些工具。

常见技能包括：

| 技能 | 说明 |
|------|------|
| **base** | 核心推理和文本生成 |
| **email** | 发送和接收邮件 |
| **file** | 读取、写入和管理文件 |
| **web_search** | 在互联网上搜索信息 |
| **agent_admin** | 管理其他智能体的配置 |
| **system_admin** | 管理系统级配置 |

## 人格 (Persona)

**人格**是智能体的性格和角色定义。它是一个文本描述，告诉智能体：

- 它是谁
- 它的职责是什么
- 它应该如何表现
- 它应该遵循什么约束

人格写在智能体的 YAML 配置文件中，在运行时加载。它们显著影响智能体对任务的响应方式。

## 邮件代理 (Email Proxy)

**邮件代理**是 AgentMatrix 和真实邮箱（通过 IMAP/SMTP）之间的可选桥梁。启用后：

- 从 AgentMatrix 发送的邮件可以投递到真实邮箱地址
- 您真实收件箱中的来件可以转发给 AgentMatrix 智能体

这允许智能体通过您的邮箱与外部联系人交互。

## 容器运行时

AgentMatrix 使用 **Podman**（首选）或 **Docker** 在隔离容器中运行智能体。这提供：

- 智能体之间的安全隔离
- 一致的运行时环境
- 资源管理

应用会自动检测您系统上可用的运行时。在 macOS 上，AgentMatrix 可以捆绑 Podman 安装程序以方便使用。
