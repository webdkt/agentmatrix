# 设置

设置面板允许您直接配置 AgentMatrix，无需编辑文件。点击视图选择器中的 **设置** 图标进入。

## 设置布局

设置面板采用两栏布局：

- **左侧边栏** —— 分类导航（LLM 配置、邮件代理）
- **右侧内容区** —— 所选分类的配置界面

## LLM 配置

LLM 配置部分管理智能体使用的 AI 模型。

### 必需配置

三种 LLM 配置是必需的，不可删除：

| 配置名称 | 用途 |
|----------|------|
| **default_llm** | 主智能体推理的主要语言模型（"大脑"） |
| **default_slm** | 用于简单任务的小语言模型（"小脑"） |
| **browser-use-llm** | 专门用于浏览器自动化的语言模型 |

### 自定义配置

您可以添加自定义 LLM 配置用于特殊用途：

- 点击 **新建** 创建自定义配置
- 自定义配置可以自由编辑和删除

### 配置字段

每个 LLM 配置需要：

| 字段 | 说明 |
|------|------|
| **配置名称** | 唯一标识符（如 "default_llm"） |
| **描述** | 人类可读的用途描述 |
| **模型名称** | 模型标识符（如 "gpt-4o"、"claude-sonnet-4-20250514"） |
| **API Key** | 您的提供商 API 密钥（本地存储） |
| **API Base URL** | API 端点 URL |

### 快速选择常用模型

配置表单包含快速选择面板，列出常用模型：

- GPT-4, GPT-3.5 Turbo
- Claude 3 Opus, Claude 3 Sonnet
- Gemini Pro

选择模型会自动填充模型名称和 API URL。

### 编辑和删除

- 点击配置卡片上的 **编辑** 按钮进行修改
- 点击自定义配置上的 **删除** 按钮进行移除
- 必需配置显示 **必需** 标签，不可删除
- 删除前会弹出确认对话框

## 邮件代理配置

邮件代理部分管理 AgentMatrix 与真实邮箱之间的连接。

### 启用 / 禁用

切换邮件代理的开关：

- **启用时**，AgentMatrix 可以通过您的真实邮箱发送和接收邮件
- **禁用时**，所有邮件通信保持在 AgentMatrix 内部

### 配置字段

| 字段 | 说明 |
|------|------|
| **邮箱地址** | 您的邮箱地址 |
| **IMAP 主机** | 收件服务器主机名（如 imap.gmail.com） |
| **IMAP 端口** | 收件服务器端口（通常为 993） |
| **IMAP 用户名** | IMAP 登录用户名 |
| **IMAP 密码** | IMAP 登录密码或应用专用密码 |
| **SMTP 主机** | 发件服务器主机名（如 smtp.gmail.com） |
| **SMTP 端口** | 发件服务器端口（通常为 587） |
| **SMTP 用户名** | SMTP 登录用户名 |
| **SMTP 密码** | SMTP 登录密码或应用专用密码 |

### 常见邮箱提供商设置

| 提供商 | IMAP 主机 | IMAP 端口 | SMTP 主机 | SMTP 端口 |
|--------|-----------|-----------|-----------|-----------|
| **Gmail** | imap.gmail.com | 993 | smtp.gmail.com | 587 |
| **Outlook** | outlook.office365.com | 993 | smtp.office365.com | 587 |
| **QQ 邮箱** | imap.qq.com | 993 | smtp.qq.com | 465 |
| **163 邮箱** | imap.163.com | 993 | smtp.163.com | 465 |

**注意**：对于 Gmail，您需要使用 [应用专用密码](https://support.google.com/accounts/answer/185833) 而非普通密码。

### 测试连接

配置邮件代理后，点击 **测试连接** 验证 AgentMatrix 能否连接到 IMAP 和 SMTP 服务器。

### 复制到 SMTP

配置表单包含 **复制到 SMTP** 按钮，可将 IMAP 设置复制到 SMTP 字段 —— 当您的提供商对两者使用相同凭据时很有用。

## 保存更改

对 LLM 配置和邮件代理设置的更改会保存到 Matrix World 配置文件中。重大更改后应用可能会提示您重启后端。
