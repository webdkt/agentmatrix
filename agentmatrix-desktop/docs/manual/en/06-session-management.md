# Session Management

Sessions are the organizational backbone of AgentMatrix — each session groups a conversation thread between you and one or more agents.

## Session List

The Session List panel is displayed on the left side of the Email View. It shows all your conversation sessions with the following information:

- **Session name or topic** — Derived from the first email subject or participants
- **Participants** — Icons or names showing who is involved in the session
- **Unread indicator** — A visual marker for sessions with new, unread emails
- **Timestamp** — The time of the last email in the session

## Browsing Sessions

- Sessions are sorted by most recent activity (newest first)
- Scroll down to load more sessions (pagination with "Load More")
- The currently selected session is highlighted

## Searching Sessions

Use the search bar at the top of the Session List to filter sessions:

- Type a keyword to match against session names and participant names
- Results update as you type
- Clear the search to see all sessions again

## Switching Sessions

Click any session in the list to switch to it. The Email List panel will update to show the emails in that session.

## Session Creation

A new session is automatically created when:

- You send a new email to an agent (not in reply to an existing session)
- An agent initiates contact with you

There is no manual "create session" button — sessions emerge naturally from communication.

## Unread Tracking

- Unread emails are visually marked with an indicator
- The session list shows unread counts or markers
- Reading an email marks it as read
- New emails trigger a notification and update the unread indicator

## Best Practices

- **Use descriptive subjects** when composing new emails — this makes sessions easier to find later
- **One topic per session** — keep conversations focused for better organization
- **Search before creating** — check if a relevant session already exists before starting a new one
