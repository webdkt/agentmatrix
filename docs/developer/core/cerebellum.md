# Cerebellum — 意图解析与参数协商

Cerebellum 是 AgentMatrix 架构中负责「格式化」的模块。它的任务是把 Brain（大模型）的自由文本输出，转换成系统可以执行的结构化动作。

---

## 设计哲学

传统 Agent 框架通常要求大模型直接输出 JSON 或特定格式的结构化数据。这带来两个问题：

1. **浪费能力**：大模型被迫同时做两件不同的事——推理和格式化。推理能力强的模型不一定擅长严格遵守输出格式。
2. **容易出错**：即使是最强的大模型，在复杂推理后输出 JSON 时也会犯语法错误（漏引号、错括号等），导致整个执行流程中断。

AgentMatrix 的解决方案是分离：

- **Brain**：只管思考，用自然语言自由表达。不需要 JSON，不需要格式约束。
- **Cerebellum**：只管解析，把自然语言翻译成动作名和参数。

这就像一个人负责想出解决方案，另一个人负责把方案写成规范的执行计划。

---

## 工作流程

### 正常路径

1. Brain 输出一段自然语言，描述它想做什么（例如"我需要先搜索一下 Python 3.13 的 release notes，然后用文件工具保存到本地"）
2. Cerebellum 读取这段文本，识别出意图和参数
3. Cerebellum 返回解析结果：动作名 + 参数字典
4. MicroAgent 用动作名在 Action Registry 中查找对应方法，传入参数执行

### 澄清路径

当 Brain 的输出比较模糊时：

1. Cerebellum 无法确定要执行哪个动作，或参数缺失/歧义
2. Cerebellum 生成一个澄清问题（例如"你想搜索哪个关键词？"）
3. 这个问题被追加到对话历史中，作为新的用户消息
4. Brain 重新思考并给出更明确的回答
5. Cerebellum 再次尝试解析

这个循环可以重复多次，直到意图足够明确或达到最大尝试次数。

---

## Cerebellum 可以由小模型担任

Cerebellum 的工作——从自然语言中提取动作名和参数——不需要强大的推理能力。一个较小的、更快的模型（甚至规则引擎）就可以胜任。

在实际部署中，你可以：

- 用同一个大模型既当 Brain 又当 Cerebellum（最简单）
- 用本地小模型（如 7B 参数模型）当 Cerebellum，降低延迟和成本
- 针对特定场景训练专用的 Cerebellum 模型

这种分离使得你可以为 Brain 和 Cerebellum 选择最适合各自任务的模型，而不需要用一个模型同时做两件事。

---

## 为什么不让 Brain 输出 JSON

有些人可能会问：如果让 Brain 直接输出 JSON 不是更简单吗？少了一个模块，少了一次 LLM 调用。

实践中我们发现：

1. **可靠性**：即使 GPT-4 级别的模型，在长推理链后输出复杂 JSON 的错误率也不低。Cerebellum 专门做这个事，可以针对格式做优化和校验。
2. **可读性**：Brain 的自然语言输出可以被人类阅读和理解。当你调试 Agent 时，看到"我打算先搜索再整理"比看到一大段 JSON 要直观得多。
3. **灵活性**：Brain 可以在思考过程中表达不确定、提出假设、进行自我纠正。这些丰富的表达如果被强制塞进 JSON 结构，会丢失大量信息。
4. **模型选择自由**：分离后，Brain 可以选用最强的推理模型，Cerebellum 可以选用最快/最便宜的模型，整体性价比更高。
