# AgentMatrix: 设计哲学

我们构建 AgentMatrix，是因为察觉到了当前智能体开发中的一个根本性错位：**我们正在用构建确定性软件的方法，去构建概率性的智能系统。**

当下的主流做法试图用严格的 JSON Schema 和硬编码的函数链去约束 LLM。这在本质上是反自然的——它强迫模糊的智能去适应精确的代码，导致系统脆弱、昂贵且难以扩展。

AgentMatrix 提出了一种新的范式：**不把 Agent 视为“功能函数”，而将其视为“组织成员”。**

### 1. 认知分层：拒绝“大材小用”
**核心观点：** 推理（Reasoning）与执行（Execution）必须在算力上物理隔离。

在传统的 Agent 中，同一个模型既要思考战略，又要去拼写 API 参数。这是对高阶算力的极大浪费。我们认为，**大脑不应该用来控制手指的每一块肌肉。**

*   **Planner/Worker (LLM)**：这是系统的**大脑**。它们只负责产生意图、逻辑和自然语言指令。它们不需要知道 API 的具体参数，只负责“想”。
*   **Secretary/PostOffice (SLM)**：这是系统的**小脑与反射弧**。它们运行在本地，由廉价的小模型驱动，专门负责将模糊的意图“翻译”为精确的系统调用、文件操作或路由分发。

这种分层让系统获得了类似生物体的鲁棒性：大脑只管决策，肌肉负责执行。

### 2. 软契约：自然语言即接口
**核心观点：** 既然 Agent 的核心是 LLM，那么 Agent 间的通信协议就应该是 LLM 的母语。

JSON 和 RPC 是机器之间的协议，它们脆弱且不容错。而在 AgentMatrix 中，**自然语言就是 IDL (接口定义语言)。**

我们放弃了“函数签名”，选择了 **Manifest (说明书)**。上游 Agent 不需要匹配下游的参数类型，只需要理解下游的“沟通偏好”。这种基于语义的**软契约 (Soft Contract)**，使得系统具有了自我修复和模糊容错的能力——这才是概率性系统的生存之道。

“邮件”不仅仅是复古，它是最适合 LLM 的**异步、带上下文、非阻塞**的通信原语。

### 3. 组织即架构：从“调用链”到“社会化”
**核心观点：** 复杂任务的解决不靠单一神话，而靠协作结构。

我们不再维护脆弱的 DAG (有向无环图) 或死板的 Chain。我们将系统建模为**人类组织**。

*   **异步 Actor 模型**：像现实公司一样，每个人（Agent）都有自己的收件箱和待办事项。大家并发工作，通过消息（邮件）交换成果，互不阻塞。
*   **路由去中心化**：发信者不需要知道“谁负责修电脑”，只要喊一声“我需要修电脑”，PostOffice（基于语义理解）就会把任务派发给最合适的角色。

### 4. 现实锚点：从“会话”到“项目”
**核心观点：** 没有文件系统支撑的智能体，只是聊天机器人。

真正的生产力产生于对物理世界（文件）的修改，而非屏幕上的字符流。AgentMatrix 强制引入 **Workspace (工作区)** 和 **Snapshot (快照)** 概念。

*   系统状态不是内存里的变量，而是磁盘上的文件。
*   所有的记忆、进度、邮件历史，都是可序列化的实体。

这意味着 AgentMatrix 的运行是可以随时**冻结、迁移和回溯**的。我们交付的不是一次对话，而是一个随时间演进的“数字工程”。

---

**总结**

AgentMatrix 试图回答一个问题：**如果操作系统是为 LLM 而非 CPU 设计的，它应该长什么样？**

答案是：它应该像一个高效的人类组织。
**分层思考，模糊通信，异步协作，结果落地。**
