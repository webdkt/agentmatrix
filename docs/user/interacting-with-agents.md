# 与 Agent 交互

AgentMatrix 的交互方式很简单：你给 Agent 发邮件，Agent 处理完回复你。但有一些技巧可以让交互更高效。

---

## 邮件即交互

在 AgentMatrix 中，所有交互都通过邮件完成。这包括：

- 你给 Agent 分配任务
- Agent 向你汇报结果
- Agent 在执行中向你提问
- Agent 之间协作完成任务

邮件由主题和正文组成。主题相当于任务的标题，正文是详细说明。Agent 会阅读整封邮件来理解你的意图。

---

## 写出有效的任务描述

Agent 理解自然语言，但清晰的描述能让它更高效地工作：

**主题要具体**。主题不需要是命令式语句，但应该让 Agent 一眼就知道任务的大致方向。好的主题比如"整理上周会议记录"、"搜索 Python 异步编程的最佳实践"。

**正文提供上下文**。如果你希望 Agent 基于某些背景信息工作，在正文中说明。例如"基于附件中的数据文件，生成一份销售趋势分析报告"。

**明确输出格式期望**。如果你希望 Agent 以特定格式输出（如 Markdown 表格、分点列表），在正文中说明。Agent 会尽量满足你的格式要求。

**一次一个主要任务**。虽然 Agent 可以处理复杂的多步骤任务，但把不同主题的事情分成多封邮件，通常能得到更好的结果。

---

## 多轮对话与上下文

Agent 会记住同一会话中的历史邮件。你可以在一封邮件中补充说明、纠正方向或追问细节。

例如，第一封邮件让 Agent"搜索 React 19 的新特性"，收到结果后，你可以回复"重点了解一下 Server Components 的变化"，Agent 会基于之前的搜索结果继续深入。

上下文是自动维护的，你不需要手动告诉 Agent"参考我上封邮件"。

---

## Agent 向你提问时

Agent 遇到以下情况时会暂停执行，向你提问：

- 任务描述不够清晰，需要 clarification
- 需要你做选择（例如"用方案 A 还是方案 B？"）
- 需要补充信息（例如"请提供 API Key"）

提问以邮件形式发送给你，桌面应用会弹出对话框通知。你可以：

- 在桌面应用中直接回复
- 如果配置了邮件代理，通过回复邮件来回答

回答后，Agent 会从断点继续执行，不需要重新启动任务。

---

## 任务与文件隔离

每个任务有独立的文件工作空间。这意味着：

- 你让 Agent A 处理「项目 A」的资料，产生的文件不会出现在「项目 B」中
- 同一个 Agent 的不同任务之间文件互不干扰
- 附件按任务隔离，你可以放心发送敏感文件

如果你需要 Agent 在不同任务之间共享文件，可以让 Agent 把文件放到一个公共位置，或者在新邮件中重新附上需要的文件。

---

## 同时与多个 Agent 协作

你可以同时向多个 Agent 发送不同的任务。每个 Agent 独立处理自己的邮件，互不干扰。

如果你希望 Agent 之间协作（例如让 Researcher 搜索资料后把结果发给 Writer 写报告），直接告诉 Writer"让 Researcher 帮我搜索某某主题的资料"。Writer 会给 Researcher 发邮件，Researcher 完成后回复 Writer，Writer 综合结果后再回复你。整个协作过程对你透明，你可以看到所有中间的邮件往来。
