# 安装与首次启动

## 环境要求

- **Python**：3.12 或更高版本
- **Node.js**：用于构建和运行 Desktop App 的前端部分
- **容器运行时**（可选）：Podman 或 Docker，用于 Agent 的隔离运行。没有容器环境时，Agent 会在本地模式运行

## 安装 Desktop App

AgentMatrix Desktop 是一个 Tauri 应用，需要同时运行 Rust 后端和 Vue 3 前端。

从源码运行的步骤：

1. 克隆仓库
2. 进入 `agentmatrix-desktop` 目录
3. 安装前端依赖
4. 启动开发模式

首次启动会自动进入冷启动向导。

## 冷启动向导

第一次运行时，系统没有配置任何 LLM，也没有创建任何 Agent。冷启动向导会引导你完成必要配置：

1. **配置 LLM** — 输入 API Key、选择模型、测试连接。支持任何 OpenAI 兼容接口的模型服务
2. **创建第一个 Agent** — 给 Agent 起名字、写一段描述、选择技能
3. **完成** — 向导关闭，进入主界面

向导完成后，你可以在设置界面随时增删 LLM 配置或修改 Agent。

## 验证安装

最简单的验证方式是向 SystemAdmin 发一封邮件，问它"显示当前系统配置"。如果收到了回复，说明 LLM 连接正常、Agent 加载正常、邮件系统正常。

## 常见问题

**启动时提示找不到容器运行时**

这是正常的。系统会回退到本地模式运行 Agent。如果你希望使用容器隔离，安装 Podman（推荐）或 Docker 后重启应用即可。

**LLM 连接测试失败**

检查 API Key 是否正确、网络是否可以访问模型服务地址、模型名称是否拼写正确。如果使用的是本地部署的模型（如 Ollama、vLLM），确认服务正在运行且端口可访问。

**界面显示为英文**

AgentMatrix Desktop 支持中英文切换。如果首次启动显示为英文，在设置中可以切换到中文。
