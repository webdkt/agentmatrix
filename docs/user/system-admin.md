# 自然语言管理系统配置

AgentMatrix 中有一个特殊的内置 Agent 叫 **SystemAdmin**，专门用来管理系统配置。你不需要手动编辑 YAML 文件，直接用自然语言告诉它你想做什么。

---

## SystemAdmin 能做什么

SystemAdmin 可以管理以下配置：

- **LLM 模型**：添加、删除、修改模型配置（API 地址、模型名称、API Key、温度参数等）
- **代理设置**：开启/关闭 HTTP 代理，修改代理地址
- **邮件代理**：配置真实邮箱的连接参数
- **容器设置**：切换本地模式与容器模式
- **系统配置查看**：展示当前所有配置的完整内容

---

## 如何使用

给 SystemAdmin 发一封邮件，用自然语言描述你的需求。例如：

> "添加一个叫 gpt4o 的模型，用 OpenAI 的 GPT-4o，API Key 是 sk-xxx"

SystemAdmin 收到后会：

1. 读取当前系统配置
2. 理解你的意图，提取关键信息
3. 验证格式是否正确
4. 测试新配置是否可用（如测试 LLM 连接）
5. 自动备份旧配置
6. 写入新配置
7. 向你汇报每一步的结果

如果某个步骤失败（例如 LLM 连接测试不通过），SystemAdmin 会停止写入，并告诉你哪里出了问题，你可以修正后再试。

---

## 示例

**添加 LLM 模型**

> "添加一个新模型，使用 Claude 3.5 Sonnet，API 地址是 https://api.anthropic.com/v1，Key 是 sk-ant-xxx"

**删除模型**

> "删除名字叫 test-model 的模型"

**开关邮件代理**

> "开启邮件代理" / "关闭邮件代理"

**查看配置**

> "显示当前系统配置" / "有哪些 LLM 模型已配置"

**修改代理**

> "设置 HTTP 代理为 http://127.0.0.1:7890"

---

## 配置版本管理

SystemAdmin 在每次修改配置前都会自动备份旧版本。备份文件存放在 MatrixWorld 的系统目录中，按时间戳命名。

如果你发现新配置有问题，可以告诉 SystemAdmin"恢复到上一个版本"或"恢复到某天的配置"。

---

## 与手动编辑的区别

| | 手动编辑配置文件 | 通过 SystemAdmin |
|--|------------------|------------------|
| 格式验证 | 容易写错 YAML 格式 | SystemAdmin 自动验证 |
| 连接测试 | 需要手动测试 | 修改前自动测试 |
| 备份 | 容易忘记 | 每次修改自动备份 |
| 回滚 | 需要手动找备份 | 一句话恢复历史版本 |
| 反馈 | 编辑完不知道对不对 | 每一步都有自然语言反馈 |

---

## 注意事项

SystemAdmin 本身也是一个 Agent，它使用 LLM 来理解你的指令。尽量用清晰、明确的语言描述需求，避免歧义。如果 SystemAdmin 理解错了，你可以纠正它，它会重新执行。

SystemAdmin 只能修改配置，不能执行危险操作（如删除文件、执行命令）。如果你需要执行系统命令，应该使用带 shell 技能的普通 Agent。
