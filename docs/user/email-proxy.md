# 邮件代理

邮件代理是 AgentMatrix 的一个可选功能。配置后，你可以用日常使用的邮箱（Gmail、Outlook、QQ 邮箱等）与 Agent 通信，不需要一直开着 Desktop App。

---

## 工作原理

邮件代理定期轮询你配置的邮箱，检查新邮件。如果邮件主题中包含 `@AgentName`（例如 `@Researcher`），邮件代理就会把邮件内容转发给对应的 Agent。Agent 处理完后，邮件代理把回复发送回你的邮箱。

整个过程对你透明：你就像在给一个同事发邮件，只不过这个同事是 AI Agent。

---

## 支持的邮箱类型

邮件代理通过 IMAP 接收邮件、通过 SMTP 发送邮件，因此理论上支持任何提供标准 IMAP/SMTP 接口的邮箱服务。

常见的支持情况：

- **Gmail**：支持，需要开启「不够安全的应用的访问权限」或使用应用专用密码
- **Outlook / Hotmail**：支持
- **QQ 邮箱**：支持，需要在邮箱设置中开启 IMAP/SMTP
- **企业邮箱**：如果提供标准 IMAP/SMTP，通常支持
- **自建邮件服务器**：支持

---

## 配置步骤

配置在 Desktop App 的设置界面中完成：

1. 打开设置 → 邮件代理
2. 填写邮箱地址、IMAP 服务器地址和端口、SMTP 服务器地址和端口
3. 填写登录密码（或应用专用密码）
4. 选择轮询间隔（多久检查一次新邮件）
5. 保存并测试连接

连接测试通过后，邮件代理即开始工作。

---

## 使用方式

### 给 Agent 发任务

新建一封邮件，主题中写上 `@AgentName`，例如：

```
主题：@Researcher 搜索 Python 3.13 的新特性
正文：整理一份清单，包括所有主要变化和对现有代码的影响
```

Agent 收到后会开始处理，处理完成后回复会发送到你的邮箱。

### Agent 向你提问

如果 Agent 在执行过程中需要向你提问，它会发一封邮件给你。你直接回复这封邮件即可，Agent 收到回复后会继续执行。

### 附件传输

你可以直接在邮件中添加附件发送给 Agent。Agent 可以读取附件内容作为任务输入。Agent 回复时如果附带文件，你也会作为邮件附件收到。

附件大小受限于你的邮箱服务商的限制。

---

## 与 Desktop App 的关系

邮件代理和 Desktop App 是并行的通信渠道：

- 通过 Desktop App 发的邮件，Agent 会处理，回复显示在 Desktop App 中
- 通过真实邮箱发的邮件，Agent 同样会处理，回复发送到你的邮箱
- Agent 不会区分消息来源，对两种渠道一视同仁

这意味着你可以在外出时用手机邮箱给 Agent 发任务，回家后在 Desktop App 里查看完整的历史记录。

---

## 安全提示

- 建议使用应用专用密码（而不是主密码）来连接邮箱
- 邮件代理只读取主题包含 `@AgentName` 的邮件，不会读取你邮箱中的其他邮件
- 邮件内容在传输过程中使用 TLS 加密
- 如果不再需要邮件代理，在设置中关闭即可
