# Desktop App 使用指南

## 界面布局

Desktop App 的主界面分为几个区域：

- **左侧边栏**：Agent 列表和状态显示。每个 Agent 显示当前状态（空闲、思考中、工作中、暂停、错误）
- **中间面板**：邮件列表和会话视图。显示你与某个 Agent 的邮件往来
- **底部输入区**：写新邮件或回复邮件
- **右侧/浮动面板**：Agent 详情、系统设置、日志等

## 写邮件给 Agent

点击界面中的「新邮件」按钮，选择收件 Agent，填写主题和正文，发送即可。

邮件主题不需要遵循特定格式，但要让 Agent 能一眼看出你想让它做什么。主题相当于任务的标题，正文是详细说明。

## 查看 Agent 的回复与执行过程

Agent 处理邮件时，状态会从「空闲」变为「思考中」，再变为「工作中」。你可以在界面上实时看到这些状态变化。

Agent 的回复以邮件形式出现在会话列表中。除了最终回复，Agent 在执行过程中产生的思考内容、调用的动作和中间结果也会以事件形式展示在界面上，让你了解它正在做什么。

## 附件

发送邮件时可以附加文件。Agent 收到邮件后，可以读取附件内容作为任务输入。Agent 回复时也可以附带文件（如生成的报告、下载的资料）。

附件按任务隔离存储，不同任务的附件不会混在一起。

## Agent 状态说明

| 状态 | 含义 |
|------|------|
| 空闲 (IDLE) | Agent 在等待新邮件 |
| 思考中 (THINKING) | Agent 正在分析任务、规划步骤 |
| 工作中 (WORKING) | Agent 正在执行动作（调用技能） |
| 恢复中 (RECOVERING) | LLM 服务中断后正在恢复 |
| 暂停 (PAUSED) | 任务被暂停，可以在任意时刻恢复 |
| 停止 (STOPPED) | 当前任务被停止，但 Agent 仍可接收新邮件 |
| 错误 (ERROR) | 执行过程中发生错误 |

## 暂停、恢复与停止

在 Agent 工作过程中，你可以随时：

- **暂停**：Agent 会在完成当前动作后停在安全检查点。暂停后可以恢复，从断点继续
- **恢复**：继续执行被暂停的任务
- **停止**：终止当前任务。Agent 回到空闲状态，可以继续接收新邮件

这些操作只影响当前正在执行的任务，不影响 Agent 接收和处理其他邮件的能力。

## 查看 Agent 的 System Prompt

在 Agent 详情面板中，可以查看该 Agent 的完整 System Prompt。这有助于理解 Agent 的行为逻辑和约束条件。

## 设置

设置界面可以管理：

- **LLM 配置**：增删模型、修改 API Key、调整参数
- **代理设置**：HTTP 代理配置
- **邮件代理**：配置真实邮箱与 Agent 交互
- **界面语言**：中英文切换
