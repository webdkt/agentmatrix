# 文档编写规范

## 核心原则

### 只写现状，不写历史
- ❌ 不要写"已废弃"、"重大变化"、"重要变更说明"等标记
- ❌ 不要进行新旧架构对比说明
- ✅ 只记录当前有效的配置、API 和使用方式

### 配置文档规范
- 必须明确区分**必须字段** vs **可选字段**
- 使用表格说明字段：字段名 | 说明 | 默认值 | 示例
- 配置示例必须完整且可直接使用

### 代码示例
- 真实可用，不要伪代码
- 标注文件路径和关键行号（如：`base.py:37`）

### 格式要求
- 使用中文标点（中文文档）
- 保持简洁，去除冗余说明
- 技术术语保持英文（如: mixin, persona, skills）
